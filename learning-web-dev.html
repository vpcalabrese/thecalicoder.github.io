Note 2: I just tested and learned that a url that goes url('images/manhattan.jpg') will link to an image, manhattan.jpg, only if the parent directory, images/, is within the same directory as the file or webpage linking it.
Prepending a slash like so url('/images/manhattan.jpg') means it doesn't matter what directory the current file is in, the slash "roots the path" of the url, or means it is a shortened form of url('https://thecalicoder.github.io/images/manhattan.jpg.

Note 3: Because Github requires a local file to clone, something like /images/ would refer to file:///images (which is the same as saying file://users/images), meaning that the file I'm linking to wouldn't be seen and I'd have to sync the website to see the results. That is why I usually type the absolute url. Get used to typing, get faster.
